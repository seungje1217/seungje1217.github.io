### Interface?

- 서로 다른 두 개의 시스템, 장치 사이에서 정보나 신호를 주고 받는 경우의 접점이나 경계면
- 다른 기기나 시스템, 소프트웨어 간에 정보를 주고받을 때, 그 사이를 잇는 규격이나 기능
- 독립되어 있는 두 개의 시스템이나 장치 간에 상호 작용 방식

이렇게 설명하면 하나도 이해가 안될것이다.
<br/>
이해하기 쉽게 인터페이스란 **무엇을 할 수 있는 지만 약속해 둔 계약서**다.
<br/>
<br/>
그럼 이게 왜 필요한걸까?
<br/>

> Interface 없는 경우
<br/>
```
class Dog {
    void sound() {
        System.out.println("멍멍");
    }
}

class Main {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.sound();
    }
}
```
아직까지는 문제가 없어 보일거다. 그게 맞는거고
<br/>
하지만 여기서 고양이 울음소리를 추가해달라는 요청이 들어오면
<br/>
```
class Cat {
    void sound() {
        System.out.println("야옹"); // 수정
    }
}

class Main {
    public static void main(String[] args) {
        Cat cat = new Cat();  // 수정
        cat.sound();
    }
}
```
이런식으로 동물들이 바뀔때마다 Main 코드를 계속 수정해야한다.
<br/>
이걸 결합도 높은 상태라고 한다 

<br/>

> Interface 있는 경우

<br/>
Interface는 아까 무엇을 할 수 있는 지만 약속해 둔 계약서라고 했다.
<br/>
즉, 계약서를 하나 만들어보자
<br/>

```
interface Animal {
    void sound();
}
```
이렇게 되면 우리가 아까 만든 Dog와 Cat은 이 계약서를 따를 것이다.
<br/>

```
class Dog implements Animal {
    public void sound() {
        System.out.println("멍멍");
    }
}

class Cat implements Animal {
    public void sound() {
        System.out.println("야옹");
    }
}
```
남은 Main은 
<br/>

```
class Main {
    public static void main(String[] args) {
        Animal animal = new Dog();
        animal.sound();
    }
}
```
이렇게 될거다.
<br/>

이제 Main은 무슨 동물인지 상관없이 sound에 관련되면 **Animal animal = new Dog();** 이 한줄만 수정하면 될것이다.
<br/>

근데 우리는 Interface가 있던 없던 결국 한줄은 수정해야하는거 아닌가라는 의문이 들수도 있다.
맞는 말이다. 
<br/>
하지만 Interface가 없을 때는 단순한 한 줄이 아니라 변수 타입, 객체 타입등 코드의 모든 Dog(우리가 바꿀 내용)에 대한 부분을 다 찾아 수정해야한다.
<br/>
<br/>
이걸 아까 말한 결합도가 높은 상태(Main이 Dog라는 클래스에 묶여있는 상태)라고 한다.
<br/>
<br/>
Interface가 있는 경우는 Animal animal = new Cat();이런 식으로 바뀌는 건 오직 **객체 생성 부분** 뿐이다. Main은 Dog가 뭔지 모를거다. Main은 Animal이라는 역할만 알 뿐..
<br/>
즉 Interface가 있는 경우에는 코드가 역할에만 묶임을 알 수 있다.

---
<br/>

Interface는 **연결 대상**에 따라 3가지로 구분할 수 있다.
<br/> 다음은 이에 대하여 알아보자

### 하드웨어 Interface
> 정의
<br/>
- 컴퓨터와 주변기기 등 하드웨어끼리 접속할때의 접점
- 서로 다른 물리적인 장치들 간의 상호작용
- 장치(부품)끼리 연결하고 데이터를 주고받기 위한 약속

> 예시

모니터와 본체를 연결하는 **HDMI**
<br/>
무선 통신 규격 **Wi-Fi, Bluetooth**
<br/>
**CPU와 RAM 사이의 버스**
<br/>
**USB**
<br/>
등 여러 종류의 하드웨어 인터페이스가 존재한다.

> 설명
<br/>

아까 인터페이스는 약속해둔 하나의 계약서라고 했는데 이거랑 무슨 연관이 있냐? 이런 의문이 들수도 있다. 
<br/>
한번 생각해보자. 만약 삼성 마우스는 삼성 전용 구멍, LG는 LG 전용 구멍 등등 각 회사마다 여러개의 구멍이 존재한다 가정하면 컴퓨터에는 적어도 20개 이상의 구멍이 필요하게 될 것이다.
세상망한다 이러면..
<br/>
그래서 다 USB로 통일하자!
<br/>
이게 바로 인터페이스다. 
<br/>
즉 하드웨어 인터페이스는 **기계(하드웨어)끼리 대화하게 만드는 공통 언어**라고 할 수 있다.


### 소프트웨어 Interface
> 정의
<br/>
- 컴퓨터를 작동시키기 위해 필요한 프로그램 간의 통신 방법
- 소프트웨어 간의 상호작용
- 프로그램과 프로그램간의 약속

> 예시

제일 쉬운 예로는 API가 있다. (API에 대해서는 추후 더 자세히 설명을 하겠다..)
이외에도 운영체제(OS)와 하드웨어들 사이의 인터페이스, 운영체제와 응용 프로그램이 상호 작용하는 인터페이스 등이 소프트웨어 인터페이스에 속한다.

> 설명


소프트웨어 인터페이스 또한 하나의 약속된 게약서다.
<br/>
우리는 어떤 웹에서 로그인 버튼을 누르면 로그인이 되는게 당연하다 생각할거다.
하지만 이게 로그인 API가 있기 때문에 가능한 것이다. 이게 바로 소프트웨어 인터페이스다.
<br/>
이게 없다고 생각해보자,,웹 마다 서버 내부 구조도 싹 다 알아야 하고, 서버 구조가 변경이라도 된다? 전부 수정해야한다. 벌써 어지럽지 않은가...
<br/>
소프트웨어 인터페이스는 내부 구조 몰라도, 약속된 형식만 알면된다.
<br/>

즉 **프로그램끼리 대화하는 공통 규칙**이다.

### 사용자 Interface (UI)
> 정의
<br/>
- 사용자와 기계 사이에서 상호작용과 정보 교환

인간이 컴퓨터나 다른 기기를 사용 시에 입력 장치나 기기, 조작 방법이나 표시 방법 등을 널리 가리키는 용어이다.

> 예시


- CLI
    - Command 창에 명령어를 입력해서 컴퓨터의 동작을 제어하는 방식
- GUI
    - 우리가 흔히 보는 화면 기반 UI (버튼, 아이콘 등)

> 설명


즉 **기능을 사람이 사용가능 하게 하는 장치**를 사용자 인터페이스라 한다. 

---
<br/>

지금까지 인터페이스를 코드 예시와 함께 살펴보았다.
<br/>

결국 인터페이스라는 단어는 어디에 쓰이느냐에 따라 모양은 달라지지만 본질은 같다.
<br/>
<br/>
하드웨어 인터페이스 → 기계 ↔ 기계
<br/>
소프트웨어 인터페이스 → 프로그램 ↔ 프로그램
<br/>
사용자 인터페이스 → 사람 ↔ 시스템
<br/>
<br/>
서로 형태는 다르지만 공통점은 하나다.
<br/>
<br/>
<br/>
**서로 다른 두 대상이 안전하게 소통하기 위한 약속**
<br/>
<br/>
결국 인터페이스는 연결을 위한 구조이자 변화를 대비한 설계다.
