어제 Flutter의 GetX에 대해 알아보았고, 오늘은 그에따른 API 통신을 하기위해 GetConnect에 대해 알아볼거다.

### API 통신?
우선 API 통신이란 웹이나 앱에서 데이터를 다루기 위해 프론트 엔드와 서버 간의 데이터 교환이 필요한데 이때 사용하는 방식을 API 통신이라고 한다.
<br/>
쉽게 말하면 클라이언트(앱)가 서버에게 데이터를 달라 or 데이터를 저장해달라고 요청(Request)하고, 서버가 그에 대한 응답(Response)을 보내주는 과정이다.
<br/>
API 통신은 클라이언트가 담당하는 UI와 서버가 담당하는 데이터 처리를 연결해주는 다리라고 생각하면 된다.

---

### Flutter + GetX의 API 통신
Flutter에서는 기본적으로 http, dio 같은 라이브러리를 사용하여 API 통신을 처리한다.
<br/>
하지만 우리는 어제 GetX를 사용하였기 때문에 GetX에서 제공하는 GetConnect를 통해 API 통신을 관리할거다.

---

### GetConnect
GetConnect는 GetX에서 제공하는 HTTP 통신 전용 클래스이다.
<br/>

UI (View)<br/>
 ↓<br/>
Controller (GetxController)<br/>
 ↓<br/>
Service / Repository<br/>
 ↓<br/>
API 통신<br/>

즉 GetX는 앱의 전체 구조를 담당하는 거고,
GetConnect는 그 안에서 API 통신을 담당한다는 거다.
<br/>

---

### GetConnect 특징

- GetX와 완전히 통합
    - Get.put(), Get.find() 그대로 사용가능
    - Controller와 자연스럽게 연결되어 구조 일관성 유지
- 별도의 네트워크 라이브러리 없이 사용가능
    - GetX 하나로 상태 관리 + 라우팅 + API 통신 처리 가능
    - 프로젝트 의존성 단슌
- CRUD API 구현에 적합
    - get, post, put, delete 메서드 제공
- Controller와 구조 분리 쉬움
    - Controller에서는 비즈니스 로직만 담당
    - 실제 HTTP 통신은 GetConnect 기반 Service에서 처리
    - 코드 가독성과 유지보수 향상

---
### 예제코드
```
import 'package:get/get.dart';

class ApiService extends GetConnect {
  @override
  void onInit() {
    httpClient.baseUrl = 'https://example.com/api';
    super.onInit();
  }

  // GET: 게시글 목록 조회
  Future<Response> fetchPosts() {
    return get('/posts');
  }

  // POST: 게시글 생성
  Future<Response> createPost(String title) {
    return post('/posts', {
      'title': title,
    });
  }
}
```

코드를 보면 extends GetConnect이런식으로 GetConnect를 상속받아 HTTP 통신 전용 클래스를 만들고,
<br/>
baseUrl만 지정해두면 이후에는 endpoint만 작성해주면 된다.
<br/>
또한 get, post 메서드로 CRUD API를 간단하게 구현할 수 있다.
